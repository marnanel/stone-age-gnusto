Installation

Don't bother downloading v0.1.0; a new release is imminent. The site is being rewritten in preparation. Check back soon!

If you're running under Unix, you must be running Mozilla as root to install add-ons. (You're adding new software, after all.) The exception is if you installed Mozilla originally as a non-root user, in which case you should run it as that user.

To install gnusto v0.2.0, click on the picture below:

Starting gnusto

Then either:

  • restart Mozilla (as an ordinary user) and choose "Interactive Fiction" from the Tools menu.
  • or start Mozilla from the command line thus:
    mozilla -chrome chrome://gnusto/content
The crash in Mozilla v1.3b

Mozilla 1.3b has a known problem with both the above methods which may crash the browser. We're looking into it. The Mozilla 1.0 branch does not have this problem. The workaround is:

  1. Restart Mozilla as an ordinary user
  2. Type the following into the location bar:
    chrome://gnusto/chrome
    You should see Gnusto load inside the Mozilla window.
  3. Close all Mozilla windows.

You can then start Gnusto as described above.

The null byte problem

Everyone will need the mz5 program until the Mozilla null byte bug is fixed. Sorry. Follow the link to find out why, and where you can get the program.

To begin your testing, you may download Dylan O'Donnell's game A Troll's Eye View in mz5 encoding from mozdev, all ready to play.

Finding stories to use

You can get other story files from the IF Archive-- try searching Baf's Guide for Z-machine games. (In the future, Gnusto will let you do this from within the program.) Please feed back on which games work and which don't by raising bugs. Currently:

Games that don't work
Curses (seems to be a parsing problem). Christminster (strange property errors)
Games that do
A Troll's Eye View. Jailbait on I-0.

Note that Gnusto currently only supports version 5. Most of these games are v5, but some may be v3 or v8. The site will tell you which.
