Installation

If you're running under Unix, you must be running Mozilla as root to install add-ons. (You're adding new software, after all.) The exception is if you installed Mozilla originally as a non-root user, in which case you should run it as that user.

To install gnusto v0.3.0, click on the picture below:

Starting gnusto

Then either:

  • restart Mozilla (as an ordinary user) and choose "Interactive Fiction" from the Tools menu.
  • or start Mozilla from the command line thus:
    mozilla -chrome chrome://gnusto/content
The crash in Mozilla v1.3

Mozilla 1.3 has a known problem with both the above methods which may crash the browser. We're looking into it. The Mozilla 1.0 branch does not have this problem. The workaround is:

  1. Restart Mozilla as an ordinary user
  2. Type this into the location bar:
    chrome://gnusto/content
    You should see Gnusto load inside the Mozilla window. (This method works on 1.3 and 1.0 branches alike.)
  3. Close all Mozilla windows.

From then on, you should be able to start Gnusto as described above. [bug ID: 3191]

The null byte problem

Everyone will need the mz5 program until the Mozilla null byte bug is fixed. Follow the link to find out why, and where you can get the program. Sorry about this-- it looks like they'll get the fix out in Mozilla v1.4. [bug ID: 3064]

To begin your testing, you may download Dylan O'Donnell's game A Troll's Eye View in mz5 encoding from mozdev, all ready to play.
